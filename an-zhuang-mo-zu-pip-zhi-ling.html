🗒️什麼是pip？

Python擁有大量的第三方函式庫，讓開發者在創造自己的程式時，也能借助前人的果實快速達成目標。
避免浪費多餘的資源做相同的事，這正是開放原始碼和共享軟體的最大宗旨。

所以既然有這麼多第三方函式庫可以使用，除了套件的集散地Python Package Index外，一款好的套件管理程式也應運而生，這就是pip模組的由來。

pip模組讓我們透過簡單的指令，從Python Package Index上下載、安裝、移除、升級任何需要的函式庫套件。
比起人為到模組的官網下載、再安裝到Python函式庫的系統位置上快速且安全的多，避免了枝微末節的錯誤、減少了錙銖必較的時間成本。

Python 3的安裝程式已經內置了pip、pip3.6、pip3三個版本，故不必再特地安裝pip模組。
可以到C:\Users\使用者名稱\AppData\Local\Programs\Python\Python36-32\Scripts查看是否有安裝好的pip模組。

🗒️pip指令

【pip3 freeze】：列出模組版本與名稱。

  1. 【pip3 list】：列出所有安裝的項目與模組。

  2. 【pip3 install 模組名稱】：安裝模組。
    pip3 install requests

    【pip3 install -v 模組名稱==版本】
    pip3 install -v requests==2.1

  3. 【pip3 uninstall 模組名稱】：移除模組。
    pip3 uninstall requests

  4. 【pip3 show --files 模組名稱】：列出模組的詳細資訊、版本、首頁、作者等。
    pip3 show --files requests

若使用蘋果的macOS作業系統，會預設安裝Python 2.7在系統之中，
如果再安裝新版的Python 3.X版，系統中等於有兩個版本Python；
下pip時需要改成pip3；如果只下pip，會導向2.7的Python中的pip。

在Windows作業系統中則無妨，若只有安裝一個版本的Python，下pip或pip3都會導向最新的Python。

🗒️安裝好的模組到哪去？

除了使用pip3 list查看安裝好的模組，如果發生了pip無法正常移除某個模組等現象，
可以到C:\Users\使用者名稱\AppData\Local\Programs\Python\Python36\Lib\site-packages查看所有安裝好的模組。
