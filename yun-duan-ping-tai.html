雲端平台

雲端平台

如果身邊沒有好的機器，好的網路環境，要架設出一部可以穩定運行的伺服器是很不容易的。
所以各家科技大廠便紛紛推出了雲端服務，除了可以架站，還能支援各式各樣的程式語言，讓開發者打造雲端應用程式。

目前最大的幾個雲端服務商，分別是：

  • Google Cloud Platform (GCP)
  • Amazon Web Service (AWS)
  • Microsoft Azure (Azure)

這些平台因為擅長的功能不同，各有各的優點，所以各自擁有一批擁護者。

🗒️連線速度

在上個章節，我們介紹過ping值，也就是一個網路封包從使用者端到伺服器端、來回一次的時間差。
也就是說，如果Ping值越低，使用者在連線時，得到資訊的速度就會越快；而讓Ping值維持很低的不二法門，就是縮短使用者端到伺服器端的物理距離。

  • GCP目前有 15 個地理區域及 45 個可用區域
  • AWS 在全球 18 個地理區域(Region)運作，並有 54 個可用區域(Availability Zones)
  • Azure 有 36 個地理區域

很幸運的是，GCP在台灣彰濱設有機房，因此對於台灣使用者來說，平均連線延遲可以降至 8ms，比 AWS 快上10倍！

🗒️運算與計價

使用這些平台，就像是遠端遙控一樣，我們傳送檔案、發布指令、部屬腳本，所有的運算壓力都在營運商上，同時，硬體的折舊、保養等消耗財，也需要營運商來負擔。

這些雲端平台，會有兩種基本的計價方式：
1️⃣一種是看你需要的運算效能。
你可以決定你的伺服器的CPU要幾核心、多大的容量、多少的Ram等。所以當然的，你需要的硬體效能越強，租借的費用就越貴。

通常這些計費方式非常靈活，每天都會依照電價進行動態調整，有時候甚至會出現離峰時段的優惠。

Heroku 雲端伺服器

上述的三種大品牌的雲端平台，雖然都有免費的試用期(通常都有12個月)、贈送等價好幾百美金的運算流量，但在申請帳號時皆需要信用卡，好讓他在試用期一道期就可以開始向你收費。

所以這裡要介紹一款深受小資開發者喜愛的雲端伺服器 - Heroku。
它可以不用信用卡就免費註冊使用，但有每天機器必須休息八小時、每30分鐘無人使用就關機等限制，不過已經十分夠用了。

  • Heroku是一個支援多種程式語言的雲平台即服務。
  • 支援Ruby、Java、Node.js、Scala、Clojure、Python、PHP和Perl等程式語言。
  • 系統底層是完全由自由軟體組成的類UNIX作業系統 - Debian
  • Heroku會提供一組與作業系統無關的特定介面。這種服務常被稱作 PaaS（platform as a service，平台即服務）。

這裡會引導大家申請一個Heroku帳號，再搭配Dropbox雲端硬碟，在平台上部屬我們的網站。

🗒️申請heroku帳號

請前往heroku官網：https://www.heroku.com/

  • 點選Sign Up For Free
  • 除了Company name不用填，其餘的依序填寫。
  • 按下確認後，heroku會發送認證信到你的email。

  • 收到後點擊正中間的連結。
  • 設定密碼
  • 出現這個Welcome字樣代表你帳號申請完成了！
🗒️部屬網站
  • 登入網站後，點選正中間的Create New App。
  • 填入你喜歡的應用名稱。
  • 選擇區域，這裡不如其他的大型雲端平台，只有美國跟歐洲兩個區域能選。完成後點擊Create app。
  • 在最上面選擇Deploy標籤，在底下找到【Deployment method部屬方法】，選擇Connect to Dropbox。
  • 如果沒有Dropbox的帳戶，請到Dropbox官網快速註冊一個。(可以直接使用Google帳戶註冊)

  • 點擊下方的Connect to Dropbox，會出現授權的提示，請點擊允許。
  • 這樣一來，heroku已經和Dropbox連線完成，並在其中新增了幾個資料夾。
  • 可以看到Conneted to ，後面接的一長串路徑指向的是會和heroku同步的資料夾。

  • 照理來說，只要我們往這個資料夾丟入html文件，連上heroku給的網址後應該就能顯示。(點我下載範例html)
  • 但heroku著重在網路應用，並不能直接運行html文件，所以我們需要使用一個php文件來引導它。
  • 請大家將你要展示的網頁檔案，改名為_index.html。
  • 打開Sublime text 3，新增一個文件，命名為index.php。並在其中加入
<?php
    include_once("_index.html");
?>
  • 這樣一來，我們透過php啟動heroku的服務，同時用函式include_once引入我們的主要網頁。
  • 把這兩個檔案丟入剛剛的Dropbox雲端資料夾中。

  • 回到我們的heroku，請點選最上面選擇Deploy標籤，在底下找到【Deploy changes部屬更新】，點擊旁邊的Deploy。
  • 如果部屬成功，Deploy按鈕下會出現3個勾勾。
  • 如果想要看你成功上線的網站，請點選下方的View按鈕

使用heroku部屬網站到這邊告一段落。

如果之後自己的網站有任何的更新，請先把它同步到Dropbox連線的資料夾中，再到heroku上點擊Deploy，簡單兩個步驟就可以完成布署了！
