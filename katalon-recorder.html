Katalon Recorder

網路應用程式是一個包含網頁的網站，其中這些網頁的內容可能有部分或全部都尚未決定。
只有在使用者向網頁伺服器要求網頁時，網頁的最後內容才會確定。
因為網頁的最後內容會根據網頁瀏覽者的動作而變化，每個要求的結果都不同，所以這種網頁便稱為動態網頁。

讓我們在看一次架構圖，比較兩者的差別：

靜態網頁

動態網頁

🗒️應用程式網站

像前一章的PChome線上購物，就是一個動態網頁，我們必須使用Selenium來開啟一個真正的瀏覽器，運行其中的JavaScript，才可以藉此得到我們所需要的HTML文件內容。
這類的網站我們依然可以從連結中發現端倪。

但還有另一種網站，使用JavaScript在本機的瀏覽器上做大部分的運算，運行起來彷彿是真正的應用程式。
比方說【網路計算機】、【高雄市公車動態網頁】。

傳統的Web應用允許用戶端填寫表單（form），當送出表單時就向網頁伺服器傳送一個請求。伺服器接收並處理傳來的表單，然後送回一個新的網頁，但這個做法浪費了許多頻寬，因為在前後兩個頁面中的大部分HTML碼往往是相同的。由於每次應用的溝通都需要向伺服器傳送請求，應用的回應時間依賴於伺服器的回應時間。這導致了使用者介面的回應比本機應用慢得多。

與此不同，網頁應用可以僅向伺服器傳送並取回必須的資料，並在用戶端採用JavaScript處理來自伺服器的回應。
因為在伺服器和瀏覽器之間交換的資料大量減少，伺服器回應更快了。
同時，很多的處理工作可以在發出請求的用戶端機器上完成，因此Web伺服器的負荷也減少了。

🗒️高雄市公車動態網頁

以這個網頁為例，我們要寫一能爬出某班公車的時刻表的程式。

使用者透過這個網站，查詢欲搭乘的公車目前所在位置以及預估到站時間。

我們進到高雄市公車動態資訊網之後，需要點擊想要搭乘的路線號碼，再點擊方向(去程/返程)。(http://ibus.tbkc.gov.tw/bus/BusRoute.aspx

就拿【0南】號公車為例好了。
到這個畫面，網頁上第一次出現完整的時刻表，包含了站序、站牌名稱、站牌代碼和預估到站時間。

但要注意的是，我們做了這麼多操作、看似送出了很多資料，網址依然不變。
這樣我們要如何讓爬蟲程式、進到該路線號碼的頁面呢？

其實做法非常直觀：模擬人類的點擊。

🗒️Katalon Recorder

  1. 什麼是Katalon Recorder？

    我們要使用的是一套叫做Katalon Recorder的瀏覽器插件。
    不管是按下按鈕或輸入文字，這套插件都可以透過元件特別的類別名稱、id，甚至是絕對的路徑，紀錄我們在該網站上的任何動作；最後將其轉換成Python程式碼，再通過Selenium的協助，重現我們在網頁上的動作。

  2. 安裝Katalon Recorder

    【點我安裝插件】

    安裝完成後，可以在網址列的右側、Chrome插件的快捷列中發現一個綠色的小圖標。
    點擊後可以打開Katalon Recorder的操作面板。

  3. 新增測試套件與測試案例

    打開操作面板後，點擊Test Suite旁的【+】新增測試套件，並將其妥善命名。
    可以命名為【高雄市公車】。

    對著測試套件按下右鍵，選擇【Add New Test Case】新增測試案例，並將其妥善命名。
    可以命名為【0南號公車】。

  4. 錄製網頁紀錄
    首先進到高雄市公車動態資訊網。
    待頁面載入完成後，打開Katalon Recorder，確定選擇【0南號公車】這個測試案例後，點擊Record按鈕。
    點擊後Katalon操作面板會自動縮小。從這裡開始，你在該網站的一舉一動都會被Katalon紀錄。

    之後我們按照使用者的使用方式，在路線的網頁點擊【0南】。
    再依照去返程，點擊對應的路徑。因為去程是預設的，為了增加複雜度，我點擊【返程】。
    在操作的過程中，我們可以在螢幕的右上角一直看到Katalon的提示。

    完成後再次點擊Katalon Recorder的操作面板，這次選擇【Stop】暫停，停止紀錄我們的操作。

  5. 指令轉成Python
    在畫面的右側可以看到的詳細操作紀錄，有【指令Command】、【目標Target】、【值Value】。
    我們可以從這略知一二，比方說open就是打開，目標是公車動態網站，所以瀏覽器的操作應該就是打開高雄市公車動態網站。
    再看到click意思是點擊，目標是【link=0南】的元素，所以瀏覽器會在HTML文件中找到【link=0】南的元素，並點擊它。

    這些指令我們都可以手動修正或新增，甚至可以按下Play按鈕，瀏覽器就會自動幫我們重複執行一次這些步驟，可以藉此來看看是否操作正確。

    在Katalon操作面板中，上方按鈕的最右側，我們找到【{ } Export】匯出按鈕。
    在format格式的清單中，我們找到【Python2(Webdriver + unittest )】。
    選擇後我們的指令就從一般的JavaScript轉成Python了！

  6. 取得Python指令
    在程式碼的欄位中，我們找到【test_0】這個函式，這裡就記錄著我們對網頁的操作，並且是使用Selenium裡的方法！

  7. 創建python檔
    打開sublime3， Ctrl + N 開啟一個新檔案， 緊接著Ctrl + S存檔， 將檔案命名為bus.py， 並且將儲存目的地選在"桌面"。(方便待會兒操作)

    匯入所需模組

    from bs4 import BeautifulSoup
from selenium import webdriver

    建立Selenium物件

    browser = webdriver.Chrome('在電腦中的位置')

    使用webdriver物件的get方法，取得網頁回應。

    browser.get('https://ecshweb.pchome.com.tw/search/v3.3/?q=airpods')

    貼上我們在test_0函式中找到的python程式碼，並記得將物件名稱改成我們自己設定的。

    browser.get("http://ibus.tbkc.gov.tw/bus/BusRoute.aspx")
browser.find_element_by_link_text(u"0南").click()
browser.find_element_by_xpath(u"(.//*[normalize-space(text()) and normalize-space(.)='捷運鹽埕埔站'])[1]/following::span[1]").click()

    使用BeautifulSoup分析，最後印出來看看吧！

    soup = BeautifulSoup(browser.page_source, "html.parser")
print(soup)

  8. 如果收不到資訊？
    有時候網路連線不佳或網路速度過慢，會發現透過Katalon的指令還是無法取得具有時間資訊的HTML文件。
